# FastAPI hitalent

Проект представляет собой API для управления информацией о сотрудниках и департаментах, реализованный с использованием FastAPI.

## Основные возможности

*   Создание, чтение, обновление и удаление (CRUD) записей о сотрудниках.
*   Создание, чтение, обновление и удаление (CRUD) записей о департаментах.
*   Проверка подлинности пользователей (аутентификация).
*   Подробное логирование всех HTTP-запросов.

## Технологии

*   **FastAPI**: Современный веб-фреймворк для создания API.
*   **SQLAlchemy**: ORM для работы с базой данных.
*   **Pydantic**: Используется для валидации данных и создания схем.
*   **Loguru**: Используется для логирования событий приложения.
*   **PostgreSQL**: Реляционная база данных, предполагается использование в качестве основного хранилища данных.

## Установка и запуск

### Способ 1: Локальный запуск

1.  Установите Python 3.12 или новее.
2.  Создайте виртуальное окружение: `python -m venv .venv`
3.  Активируйте виртуальное окружение:
    *   Windows: `.venv\Scripts\activate.bat`
    *   Linux/macOS: `source .venv/bin/activate`
4.  Установите зависимости: `pip install -r requirements.txt`
5.  Настройте переменные окружения, создав файл `.env` на основе примера `.env.example`. Необходимые переменные включают строки подключения к базе данных.
6.  Запустите сервер: `uvicorn main:app --reload`
7.  Документация API будет доступна по адресу `http://127.0.0.1:8000/docs`.

### Способ 2: Запуск через Docker

1.  Убедитесь, что у вас установлены Docker и docker-compose.
2.  Запустите команду: `docker-compose up --build`
3.  Приложение и база данных будут автоматически запущены. Документация API будет доступна по адресу `http://127.0.0.1:8000/docs`.

## Логирование

Приложение использует `Loguru` для логирования. Логи всех запросов записываются в файл `info.log` с уникальным идентификатором для каждого запроса. Мидлварь `log_middleware` автоматически:

*   Генерирует уникальный `log_id` для каждого запроса.
*   Логирует успешные запросы.
*   Логирует предупреждения при ошибках 4xx (например, 404 Not Found).
*   Логирует критические ошибки при исключениях.

## Структура проекта

*   `main.py`: Точка входа в приложение.
*   `routers/`: Маршруты API.
*   `models/`: Модели SQLAlchemy.
*   `schemas.py`: Модели Pydantic для валидации данных.
*   `crud.py`: Функции для работы с базой данных (CRUD операции).
*   `logger_config.py`: Настройка логирования и мидлварь.
*   `database.py`: Настройка подключения к базе данных.
*   `db_depends.py`: Зависимости для FastAPI (например, получение сессии базы данных).
*   `requirements.txt`: Список зависимостей Python.
*   `migrations/`: Скрипты миграций базы данных (Alembic).
